# PREMO Review System

## Project Goal

This project recreates the existing PREMO Review System.

Primary objective:

- Match the existing website as closely as possible.
- Pixel-perfect recreation.
- Same layout.
- Same spacing.
- Same colors.
- Same typography.
- Same icons.
- Same border radius.
- Same shadows.
- Same responsive behavior.

This is NOT a redesign project.

---

# Development Rules

DO NOT:

- redesign anything
- improve the UI
- modernize components
- simplify layouts
- replace components with better alternatives
- change spacing
- change colors
- change fonts
- change border radius
- change shadows
- change animations
- change icon styles
- change user flow

Always copy the original website exactly.

---

# Tech Stack

- Next.js App Router
- TypeScript
- Tailwind CSS
- shadcn/ui

---

# Development Priority

Priority order:

1. UI
2. Responsive Layout
3. Interactions
4. Backend
5. Optimization

Backend will be implemented later.

Focus on UI first.

---

# Coding Style

Keep code:

- simple
- reusable
- clean
- readable

Avoid unnecessary abstraction.

---

# Components

Reuse components whenever possible.

Create shared components for:

- cards
- forms
- tables
- buttons
- dialogs
- navigation
- badges
- tabs

---

# Page Rules

Every page should be built independently.

Do not modify existing pages unless requested.

---

# Images

Use placeholder images if assets are unavailable.

Do not redesign missing assets.

---

# Icons

Use Lucide icons unless the original asset is available.

---

# Responsive

Desktop first.

Then mobile.

Match original responsive behavior.

---

# When screenshots are provided

Treat screenshots as the source of truth.

Always reproduce them as accurately as possible.

Never redesign based on personal preference.

---

# Backend

No backend unless explicitly requested.

UI first.

---

# Git

Keep commits small.

Do not rename files unnecessarily.

Do not restructure folders unless required.

---

# Workflow

Always complete work in this order:

1. Recreate UI
2. Ensure visual match
3. Fix runtime errors
4. Verify responsive layout
5. Commit changes

Never skip runtime verification.

Always ensure the project builds successfully before finishing.

---

# Rule Zero

If there is any conflict:

Copy the original website exactly.

Do not improve it.
